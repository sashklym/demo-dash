# Frontend — AI Coding Rules

React 18 + Vite + Tailwind + shadcn/ui + Recharts + React Query. Full conventions in [`.claude/skills/react-frontend`](../.claude/skills/react-frontend/SKILL.md).

## Structure

```
src/
├── main.tsx           providers (QueryClient, Router, Toaster)
├── router.tsx         "/" (bootstrap) + "/d/:key"
├── routes/            BootstrapRoute, DashboardPage
├── components/        DashboardHeader, WidgetGrid, WidgetCard, ChartWidget, TextWidget, AddWidgetMenu
│   └── ui/            shadcn primitives (editable)
├── hooks/             use-widgets (wrappers over generated hooks + cache invalidation)
└── lib/api/           http (axios), openapi.json, generated/ (orval output — do not edit)
```

## Forbidden

- Hand-written `fetch`/`axios` → use the generated hooks via `hooks/use-widgets.ts`.
- Editing `src/lib/api/generated/` → it's regenerated by `npm run api:generate`.
- `mutate(vars, { onSuccess })` for **mount-triggered** mutations → use `mutateAsync().then(...)` (StrictMode discards per-call callbacks on unmount). Click-triggered mutations may use per-call callbacks.
- Faking per-widget loading/error → each chart runs its own `useGetWidgetData` query.
- `any`.

## Types

Import `Widget`, `WidgetType`, etc. from `@/lib/api/generated/model`. The `@/` alias maps to `src/`.

## Verification gate (before commit)

```bash
npm run api:generate && npm run lint && npm run typecheck \
  && npm run test:unit && npm run build
```
